Live Protocol Proof

cx402 has five proof modules. They are here so you do not have to trust the UI. Each one verifies a different part of the agent-payment trust model.

Protected Agent API
/premium

This is the seller side. A data API, MCP tool, or agent service can sit behind cx402. If an agent requests the resource without payment, it receives a 402 Payment Required challenge. If it sends a valid compliant payment, the resource is served with a receipt.

Open raw 402 challenge
Signed Agent Mandate
/proof/mandate

This is the operator side. The human or institution gives the agent a signed spending mandate: budget, max payment, minimum counterparty tier, and optional allow-list. cx402 rejects forged, expired, tampered, or replayed mandates.

Open proof
Signed Payment Intent
/proof/payment-intent

This is the per-payment authorization layer. A payment intent binds the payer to the exact payee, amount, asset, network, resource, nonce, and expiry. If any field changes, the signature breaks.

This proves the demo is live: Cleanverse is reachable, Monad is reachable, both parties have valid A-Pass code 4, and settlement uses real aUSDC.

How Payments Get Cleared or Blocked

Both parties verified

Every payment is checked against the live Cleanverse A-Pass registry. No valid A-Pass on either side, no settlement.

Blocked before funds move

cx402 checks Cleanverse A-Pass before settlement, so non-verified counterparties are blocked before funds move.

Within budget and rules

Budget, per-tx cap, min-tier, allowed counterparties, set by the operator and enforced across payments. The budget depletes.

Two Integrations: Buyer and Seller

cx402 is something other builders drop into what they're making. There are two kinds of user, and each gets the whole compliance layer for almost no work. Everything underneath, the identity checks, the settlement, the receipts, is handled for them.

for the buyer · whoever runs the agent

Give your AI agent a mandate

Hand it a wallet and a mandate. The agent pays suppliers on its own, and cx402 refuses anything that breaks a rule or pays an unverified seller. The rules you can set:

  • spending budget depletes as it pays
  • max per payment per-payment cap
  • minimum counterparty tier required KYC level
  • allowed counterparties verified, or an allow-list
const agent = cx402.agent({ wallet, policy }) await agent.pay({ payee, amount })
all four enforced on every payment.
for the seller · whoever gets paid

Only get paid by verified buyers

Put one line in front of your API, your service, or your agent. From then on it only accepts payments from verified, in-policy counterparties, each one already settled on-chain and backed by a receipt. You never have to trust the buyer.

app.use('/api', cx402Paywall({ price, payTo }))
one line, fully protected.
